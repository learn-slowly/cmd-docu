# Phase 8.5 (단계 ③) 폴더별 뷰 기억 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 라이브러리 레이아웃(list/grid)을 폴더별로 기억해 런치 간 복원한다(미기억 폴더는 현재 유지).

**Architecture:** `AppSettings.libraryLayouts: [String: LibraryLayout]`(폴더경로→layout)를 settings.json에 영속. `AppState`의 `selectedFolder`·`libraryLayout`에 `didSet`을 달아, 폴더가 바뀌면 기억된 layout을 복원하고(없으면 유지) 사용자가 layout을 바꾸면 현재 폴더에 저장+`saveUserData()`한다. 복원 중 중복 저장은 `isRestoringLayout` 플래그로 막는다.

**Tech Stack:** Swift 5.9 / SwiftUI(@Observable + didSet), Codable, XCTest.

## Global Constraints

- macOS 14+, Swift 5.9+, SPM. 비샌드박스.
- 코드 주석·커밋 메시지 **한국어**. '박다/박는다/박았다' 금지.

```swift
import XCTest
@testable import CmdMD

@MainActor
final class AppLibraryLayoutMemoryTests: XCTestCase {
    /// AppState()는 세션에서 실제 currentFolder를 로드할 수 있다. 실제 사용자 설정을
    /// 오염시키지 않도록 폴더를 비우고 가짜 경로만 쓴다.
    private func freshState() -> AppState {
        let s = AppState()
        s.currentFolder = nil
        s.selectedFolder = nil
        s.settings.libraryLayouts = [:]
        return s
    }

    func testUnrememberedFolderKeepsCurrentLayout() {
        let s = freshState()
        s.selectedFolder = URL(fileURLWithPath: "/v/x")  // 폴더부터 지정(persist 대상 명확)
        s.libraryLayout = .list                           // /v/x에 list 저장
        s.selectedFolder = URL(fileURLWithPath: "/v/unknown")  // 미기억 폴더
        XCTAssertEqual(s.libraryLayout, .list)            // 유지(리셋 안 함)
    }

    func testRemembersAndRestoresPerFolder() {
        let s = freshState()
        s.settings.libraryLayouts["/v/photos"] = .grid
        s.selectedFolder = URL(fileURLWithPath: "/v/docs")
        s.libraryLayout = .list                           // /v/docs=list
        s.selectedFolder = URL(fileURLWithPath: "/v/photos")
        XCTAssertEqual(s.libraryLayout, .grid)            // 기억된 layout 복원
    }

    func testTogglingLayoutPersistsForCurrentFolder() {
        let s = freshState()
        s.selectedFolder = URL(fileURLWithPath: "/v/docs")
        s.libraryLayout = .list                           // 사용자가 토글
        XCTAssertEqual(s.settings.libraryLayouts["/v/docs"], .list)
    }

    func testRestoreDoesNotCreateExtraKeys() {
        let s = freshState()
        s.settings.libraryLayouts = ["/v/photos": .grid]
        s.selectedFolder = URL(fileURLWithPath: "/v/photos")  // 복원만(저장 유발 X)
        XCTAssertEqual(s.settings.libraryLayouts, ["/v/photos": .grid])
    }
}
```

- [ ] **Step 3: 구현 — didSet + 헬퍼 + 플래그**

`Sources/App/AppState.swift`의 선언(~31-33)을 교체:
```swift
    /// 라이브러리 뷰가 보여줄 폴더. 기본·리셋값은 currentFolder.
    var selectedFolder: URL? = nil {
        didSet { restoreLibraryLayoutForSelectedFolder() }
    }
    /// 라이브러리 뷰 레이아웃(grid/list). 폴더별로 기억된다(8.5-③).
    var libraryLayout: LibraryLayout = .grid {
        didSet { persistLibraryLayoutForCurrentFolder(oldValue: oldValue) }
    }
    /// 폴더 전환에 따른 layout 복원 중에는 저장을 막는다(되먹임·중복 저장 방지).
    private var isRestoringLayout = false
```

그리고 메서드 추가(예: 라이브러리 관련 메서드 부근):
```swift
    /// selectedFolder에 기억된 레이아웃이 있으면 복원한다(없으면 현재 유지).
    private func restoreLibraryLayoutForSelectedFolder() {
        guard let folder = selectedFolder else { return }
        let key = folder.standardizedFileURL.path
        guard let remembered = settings.libraryLayouts[key], remembered != libraryLayout else { return }
        isRestoringLayout = true
        libraryLayout = remembered
        isRestoringLayout = false
    }

    /// 현재 폴더에 대해 layout 선택을 기억·영속한다. 복원 중이면 스킵.
    private func persistLibraryLayoutForCurrentFolder(oldValue: LibraryLayout) {
        guard !isRestoringLayout, libraryLayout != oldValue,
              let folder = selectedFolder ?? currentFolder else { return }
        let key = folder.standardizedFileURL.path
        guard settings.libraryLayouts[key] != libraryLayout else { return }
        settings.libraryLayouts[key] = libraryLayout
        saveUserData()
    }
```

> @Observable 저장 프로퍼티 + didSet은 Swift Observation에서 지원된다(관찰 추적과 무관하게 setter에 보존). 만약 빌드가 didSet을 거부하면(드묾), 해당 프로퍼티를 `@ObservationIgnored` 백킹 + 명시 get/set으로 감싸는 대안이 있으나 먼저 didSet으로 시도한다.
