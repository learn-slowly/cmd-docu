# Phase 8.5 (단계 ②b) — 라이브러리 그리드 썸네일 설계

> 작성일: 2026-07-01 · 대상: cmd-docu · 우선순위: 티어 3 (8.5-②a 다음)
> 관련: PRD §3.9·Phase 8.5, 선행 8.5-②a(LibraryView/모드토글) 완료.
> 범위: 라이브러리 **그리드** 셀의 종류별 SF 아이콘을 실제 **썸네일**로 교체(QuickLook). 리스트는 손대지 않음.

## 1. 목적

라이브러리 그리드에서 파일을 "재생/열기 전에" 알아보게 — 이미지·PDF·영상·문서의 실제
미리보기 썸네일을 셀에 보여준다. 못 만드는 타입은 기존 아이콘으로 폴백. 읽기 전용(생성만).

## 2. 확정된 결정 (brainstorming)

- **대상**: 전 파일에 QLThumbnailGenerator 시도 + **실패/미지원이면 SF 아이콘 폴백**. 폴더는 폴더 아이콘 유지(썸네일 없음).
- **적용 뷰**: **그리드만**. 리스트는 기존 작은 SF 아이콘 유지.
- **생성 시점**: 셀이 화면에 들어올 때 비동기 1회(lazy), 사라지면 취소. NSCache로 재사용.
- **무효화**: 캐시 키에 mtime 포함 → 파일 편집 시 자동 갱신.
- **경계**: 읽기 전용 — 파일 이동·삭제·변경 없음. 생성만.

## 3. 컴포넌트

### 신규 `Sources/Services/ThumbnailService.swift`
- `import QuickLookThumbnailing`(macOS 내장; 비샌드박스라 파일 접근 OK).
- `@MainActor final class ThumbnailService` + `static let shared`.
  - `private let cache = NSCache<NSString, NSImage>()`
  - `func thumbnail(for url: URL, pointSize: CGFloat, scale: CGFloat) async -> NSImage?`
    - 캐시 키 조회 → 히트면 즉시 반환.
    - 미스면 `QLThumbnailGenerator.Request(fileAt: url, size: CGSize(pointSize×scale), scale: scale, representationTypes: .thumbnail)` →
      `QLThumbnailGenerator.shared.generateBestRepresentation(for:)` → `NSImage(cgImage:)` → 캐시 저장 → 반환.
    - 실패/미지원/접근불가 → `nil`(셀이 아이콘 폴백). 크래시 금지(throw 잡아 nil).
  - `static func cacheKey(url: URL, pointSize: CGFloat, scale: CGFloat, mtime: TimeInterval) -> String` — **순수**.
    `"\(url.path)|\(pointSize)|\(scale)|\(mtime)"`. mtime은 호출부에서 resourceValues로 읽어 전달(없으면 0).

### 수정 `Sources/Views/LibraryView.swift` — `LibraryGridCell`
- `@State private var thumbnail: NSImage?` 추가.
- 이미지 영역(현재 `Image(systemName: cellIcon).font(.system(size: 32))`)을 고정 크기 박스로:
  - 파일이고 `thumbnail != nil` → `Image(nsImage:).resizable().scaledToFit().frame(width: 64, height: 64)`.
  - 아니면(폴더이거나 아직 로드 전·폴백) → 기존 SF 아이콘(동일 위치·크기 유지).
- `.task(id: item.url)`에서 파일이면 `thumbnail = await ThumbnailService.shared.thumbnail(for: item.url, pointSize: 64, scale: 2)`.
  - `item.isDirectory`면 생성 안 함(폴더 아이콘 유지).
  - mtime은 task 안에서 `item.url.resourceValues(forKeys: [.contentModificationDateKey])`로 읽어 서비스에 넘기거나, 서비스가 내부에서 읽음(서비스가 읽도록 한다 — 호출부 단순화).
- archive dim·이름·탭 제스처·`paraCategory` 스타일은 그대로.

> 폴더 아이콘 강조(projects=accent)·archive dim은 썸네일이 떠도 유지(셀 컨테이너 레벨 opacity).

## 4. 데이터 흐름

```
LazyVGrid → (보이는 셀만) LibraryGridCell
  .task(id: url): if !isDirectory → ThumbnailService.shared.thumbnail(url, 64, 2)
      → cacheKey(url,size,scale,mtime) 조회
         히트 → NSImage 반환
         미스 → QLThumbnailGenerator 생성 → 캐시 → 반환 (실패 시 nil)
  → thumbnail != nil ? Image(nsImage) : SF 아이콘(폴백)
셀이 스크롤 아웃 → SwiftUI가 .task 취소(생성 중단)  ← "화면 밖 보류"
```

## 5. 에러·성능

- QL 실패(미지원 타입 HWP 등·손상·접근불가) → nil → 아이콘 폴백. 무음 처리, 크래시 없음.
- LazyVGrid가 보이는 셀만 인스턴스화 → 1000+ 폴더에서도 동시 생성 폭주 없음. `.task` 취소로 스크롤 시 불필요 생성 중단.
- NSCache는 메모리 압박 시 자동 비움(별도 만료 불필요).

## 6. 테스트 전략

- `ThumbnailServiceTests` — `cacheKey(url:pointSize:scale:mtime:)` 순수성: 같은 입력 같은 키, mtime/size/scale 바뀌면 키 변경, url 다르면 키 다름. (실제 QL 생성·NSCache·셀 렌더는 환경/UI 의존 → 빌드+수동.)
- 수동: 이미지·PDF·영상 폴더 그리드에서 썸네일 표시, HWP·md는 아이콘 폴백, 스크롤·드릴인 시 멈춤 없음, 같은 폴더 재방문 시 캐시로 즉시.
