# Phase 8.5 (단계 ①) — 사이드바 PARA 렌즈 설계

> 작성일: 2026-06-30 · 대상: cmd-docu · 우선순위: 티어 3 (Phase 8 다음, 9 앞)
> 관련: PRD §3.9 (PARA 라이브러리 뷰), CLAUDE.md Phase 8.5
> 범위: Phase 8.5의 **첫 서브프로젝트(단계 ①)만**. 메인 라이브러리 그리드·모드토글은 단계 ②, 폴더별 뷰 기억은 단계 ③로 분리.

## 1. 목적

파일 트리(사이드바)를 PARA 구조로 "렌즈"를 끼워 본다. 폴더를 PARA 분류
(Projects/Areas/Resources/Archive)대로 정렬하고, 분류별 시각 위계를 입혀
"즉시 PARA 느낌"을 준다. 가장 가벼운 슬라이스 — 읽기/탐색 전용, 데이터·상태 불변.

## 2. 확정된 결정 (brainstorming)

- **표현**: 평평한 트리 유지 + **정렬·스타일만**. 섹션 헤더·별도 토글 없음.
- **분류 판별**: **숫자 접두사 규칙** — 폴더명 접두사 `10000_→Projects`,
  `20000_→Areas`, `30000_→Resources`, `40000_→Archive`. 없으면 `other`.
  (`legoSeed` 체계와 일치, 설정·볼트 구성과 무관하게 동작.)
- **적용 범위**: **최상위 포함 캐스케이드** — 분류는 경로의 최상위 PARA 루트로
  결정·상속. `40000_Archive` 하위 모든 항목 dim, `10000_Projects` 하위 강조 유지.
- **경계**: 읽기/탐색 전용. 파일 이동·삭제·이름변경 없음. 새 영속 상태·토글 없음.

## 3. 접근법 — 순수 표현 헬퍼 (`ParaLens`)

기존 `RouteHelper`·`CleanupPlanner` 패턴을 따라 **순수 헬퍼 모듈** 하나를 두고,
`FileTreeItem` 데이터 모델과 `AppState`는 건드리지 않는다. 정렬·스타일은
**렌더 시점에만** 적용한다. (대안 1: 모델에 category 필드 추가 — 모델 침투라
기각. 대안 2: Files⇄PARA 토글 — 사용자가 "평평·항상 스타일"을 택해 불필요.)

PARA 폴더가 없는 일반 폴더를 열면 모든 항목이 `other`로 분류돼 화면 변화가
없다 → 별도 트리거 판정이 필요 없다(안전).

## 4. 컴포넌트

### 신규: `Sources/Services/ParaLens.swift` (순수, 테스트 대상)

- `enum ParaCategory: Equatable { case projects, areas, resources, archive, other }`
  - `var sortRank: Int` — projects 0 · areas 1 · resources 2 · other 3 · **archive 4(맨 끝)**.
- `static func classify(_ url: URL) -> ParaCategory`
  - `url.pathComponents`를 **앞에서부터(최상위→하위)** 스캔, 처음으로
    `^[1-4]0000_` 접두사에 매칭되는 컴포넌트로 분류. 매칭 숫자 → 분류 매핑.
    매칭 없으면 `.other`. (캐스케이드: 최상위 PARA 루트가 하위 전체를 결정.)
- `static func sorted(_ items: [FileTreeItem]) -> [FileTreeItem]`
  - 정렬키: `(category.sortRank, isDirectory ? 0 : 1, name localizedStandard)`.
    같은 분류 안에서 폴더 먼저, 이름 오름차순. 기존 항목 식별자·children 보존.

### 수정: `Sources/Views/SidebarView.swift` (렌더만)

- `FileTreeView`: 루트 `ForEach(appState.fileTree)` → `ForEach(ParaLens.sorted(appState.fileTree))`.
- `FileTreeItemRow`: 자식 `ForEach(item.children)` → `ForEach(ParaLens.sorted(item.children))`.
- 행 스타일은 `ParaLens.classify(item.url)` 결과로 분기(아래 §6). 폴더/파일 두 분기
  모두에 적용. 즐겨찾기 별·컨텍스트 메뉴·펼치기(DisclosureGroup) 동작은 보존.

## 5. 데이터 흐름

```
loadFileTree (기존, 변경 없음) → appState.fileTree: [FileTreeItem]
  → FileTreeView: ForEach(ParaLens.sorted(fileTree))           # 루트 정렬
      → FileTreeItemRow(item): classify(item.url) → 스타일
          → DisclosureGroup 펼침 시 ForEach(ParaLens.sorted(item.children))  # 재귀 정렬
```

데이터는 정렬·분류와 무관하게 그대로. 렌즈는 순수 함수의 표현 계층.

## 6. 스타일 규칙

- **archive**: 행 전체 `.opacity(0.45)` — 차분히 가라앉힘.
- **projects**: 폴더 아이콘 `.foregroundStyle(.cmdsAccent)` + 이름 `.fontWeight(.medium)` — 또렷.
- **areas / resources / other**: 기본 스타일(변경 없음).

스타일은 `FileTreeItemRow`의 `Label`/행에 적용. 분류는 `item.url` 기준이라
캐스케이드로 archive 하위 항목도 자연히 dim.

## 7. 테스트 전략

`ParaLensTests` (순수 in-process 단위테스트):
- `classify`:
  - `…/10000_Projects` → projects, `…/20000_Areas` → areas,
    `…/30000_Resources` → resources, `…/40000_Archive` → archive
  - 중첩 캐스케이드: `…/10000_Projects/Living_with_Damage/note.md` → projects,
    `…/40000_Archive/2024/old.pdf` → archive
  - 접두사 없는 경로 → other
  - 최상위 우선: 경로에 PARA 루트가 있으면 더 깊은 비매칭과 무관하게 그 분류
- `sorted`:
  - 루트 4개가 Projects→Areas→Resources→(기타)→Archive 순, archive 맨 끝
  - 같은 분류 내 폴더 먼저, 이름 오름차순
  - 식별자·children 보존(정렬이 항목 내용 손상 없음)

UI(스타일 표시·정렬 반영)는 수동 검증(사이드바 육안 + Phase 게이트).

## 8. Phase 게이트

- 시작 전: `swift test` 기준선(현 212개) 통과 확인.
- 완료 후: 기존 + `ParaLensTests` 통과. UI는 실제 PARA 볼트로 수동 확인
  (Projects 또렷·Archive dim·정렬). `swift test`엔 정식 Xcode 필요.

## 9. 범위 밖 (후속 단계/YAGNI)

- 썸네일(QLThumbnailGenerator), 폴더 선택(`selectedFolder`) 개념 — 단계 ②.
