# Finder 대체 로드맵 (확정)

- 날짜: 2026-07-03
- 상태: 사용자 확정(대화). 각 조각은 별도 spec→plan→구현 사이클.
- 방향(사용자 결정): **일상 파일작업 대체** — 브라우징·복사·이동·이름변경·새폴더·휴지통·다중선택·정보보기를 cmdALL에서 처리해 평소 작업에서 Finder 열 일을 없앤다. 외부 볼륨·네트워크·데스크톱 관리·시스템 전역 대체는 범위 밖.
- 삭제 정책(사용자 결정, 기존 "삭제 없음" 규칙의 공식 개정): **휴지통 이동만 + 작업 로그** — `FileManager.trashItem`으로 휴지통 이동(Finder와 동일하게 복구 가능), 이동 내역을 작업 로그에 기록해 앱 안에서 되돌리기 지원. 영구 삭제·휴지통 비우기는 만들지 않는다.

## 조각과 순서 (F1a → F1b → F3 → F2 → F4)

| 조각 | 내용 | 상태 |
|---|---|---|
| **F1a. 파일 작업 기반** | 이름 변경·새 폴더·휴지통(단일 항목, 트리·라이브러리 컨텍스트 메뉴) + `FileOperations` 서비스 + 작업 로그(휴지통·이름변경 기록, 앱 내 되돌리기) + **정보 보기**(⌥⌘I 시트 — ⌘I는 Italic 선점 — + 라이브러리 리스트 크기·수정일 열; 2026-07-03 F3에서 앞당김) | 설계 중 |
| **F1b. 다중 선택 + 배치** | 라이브러리·트리 ⌘/⇧클릭 다중 선택, 일괄 이동·복사·휴지통, ⌘C/⌘V 복사·붙여넣기 | 대기 |
| **F3. 탐색 강화** | 경로 바(브레드크럼)·뒤로/앞으로 히스토리·정렬 옵션(이름/날짜/크기/종류 — 리스트 열 헤더 클릭 정렬 포함) | 대기 |
| **F2. 드래그 이동** | 라이브러리·트리 안팎 드래그&드롭(Finder 양방향), ⌥=복사 | 대기 |
| **F4. 듀얼 페인** | 두 폴더 나란히 — 이동·복사의 무대, 파일 정리 완성형 | 대기 |

## 공통 원칙

- 파일 작업은 전부 로그를 남기고 앱 안에서 되돌릴 수 있게(기존 정리 배치 undo 관례 확장).
- 덮어쓰기 금지·충돌 시 uniquify(기존 MoveExecutor 관례).
- 열린 탭·검색 인덱스·파일 트리와의 정합을 각 조각 스펙에서 명시.
- 기존 리더·검색·Claude 기능은 불변 — 파일 관리 층은 별도 파일/모듈로 가산.
