# Phase 8.5 (단계 ③) — 폴더별 뷰 기억 설계

> 작성일: 2026-07-01 · 대상: cmd-docu · 우선순위: 티어 3 (8.5-②b 다음, 8.5 마지막 단계)
> 관련: PRD §3.9·Phase 8.5. 선행: ②a(LibraryView/모드토글)·②b(썸네일) 완료.
> 범위: 라이브러리 레이아웃(list/grid)을 **폴더별로 기억**해 런치 간 복원. 이걸로 Phase 8.5 완결.

## 1. 목적

폴더마다 보던 방식(리스트/그리드)을 기억한다. 사진 폴더는 그리드로, 문서 폴더는 리스트로
두면 다음에 그 폴더를 열 때 그대로 복원된다. 앱을 껐다 켜도 유지. 읽기/탐색 전용.

## 2. 확정된 결정 (brainstorming)

- **영속 위치**: **AppSettings**(settings.json, 런치 간 유지). `[폴더경로: LibraryLayout]`.
- **미기억 폴더**: **현재 레이아웃 유지**(grid 강제 리셋 안 함). 기억된 폴더를 만나면 그때 복원.
- **저장 시점**: 사용자가 툴바에서 layout을 바꿀 때 현재 폴더에 기억.
- **복원 시점**: selectedFolder가 바뀔 때(드릴인·상위·새 볼트·세션 복원), 그 폴더에 기억이 있으면 복원.
- **경계**: 표시 레이아웃(list/grid)만 기억. 파일 변경 없음.

## 3. 컴포넌트

### 수정 `Sources/Models/Settings.swift` — `AppSettings`
- 필드 추가: `var libraryLayouts: [String: LibraryLayout] = [:]` (키=폴더 표준화 경로, 값=list/grid).
- 하위호환 디코드: `init(from:)`에 `libraryLayouts = try c.decodeIfPresent([String: LibraryLayout].self, forKey: .libraryLayouts) ?? d.libraryLayouts` (기존 패턴). CodingKeys에 추가.
- `LibraryLayout`은 `String, Codable`이라 `[String: LibraryLayout]`은 곧장 Codable.

### 수정 `Sources/App/AppState.swift`
- `private var isRestoringLayout = false` (복원 중 중복 저장 억제).
- `selectedFolder` 에 `didSet` 추가:
  ```swift
  var selectedFolder: URL? {
      didSet { restoreLibraryLayoutForSelectedFolder() }
  }
  ```
  `restoreLibraryLayoutForSelectedFolder()` — selectedFolder의 표준화 경로 키로 `settings.libraryLayouts`에 값이 있으면 `isRestoringLayout = true` 상태로 `libraryLayout`에 대입(없으면 아무것도 안 함=현재 유지).
- `libraryLayout` 에 `didSet` 추가:
  ```swift
  var libraryLayout: LibraryLayout = .grid {
      didSet { persistLibraryLayoutForCurrentFolder(oldValue: oldValue) }
  }
  ```
  `persistLibraryLayoutForCurrentFolder(oldValue:)` — 복원 중(`isRestoringLayout`)이면 스킵, 값이 안 바뀌었으면 스킵. 아니면 현재 폴더(`selectedFolder ?? currentFolder`)의 키에 `settings.libraryLayouts[key] = libraryLayout` 저장. (settings 변경 → 기존 `onChange(of: settings) { saveUserData() }`가 영속.)

> 모든 selectedFolder 쓰기 지점(LibraryView 드릴인·goUp·`selectFolderForLibrary`·openFolder/세션 복원의 currentFolder 리셋)이 자동으로 didSet을 타므로 호출부 수정 불필요. @Observable 저장 프로퍼티 + didSet은 Swift Observation에서 지원.

## 4. 데이터 흐름

```
[폴더 변경] selectedFolder= … → didSet → restoreLibraryLayoutForSelectedFolder()
    settings.libraryLayouts[key] 있으면 → isRestoringLayout=true; libraryLayout=remembered; isRestoringLayout=false
    없으면 → no-op(현재 layout 유지)
[토글] libraryLayout= … (툴바 Picker) → didSet → persistLibraryLayoutForCurrentFolder()
    isRestoringLayout이면 skip; 아니면 settings.libraryLayouts[현재폴더 key]=libraryLayout → settings 변경 → saveUserData
[재시작] 세션 복원: currentFolder/selectedFolder 설정 → didSet이 기억 복원
```

키 = `url.standardizedFileURL.path`(일관성·중복 방지).

## 5. 에러·엣지

- selectedFolder == nil → 복원 no-op.
- 같은 값으로의 복원/토글 → didSet 가드(`oldValue` 비교)로 불필요 저장 방지.
- 복원이 트리거한 libraryLayout didSet → `isRestoringLayout` 플래그로 저장 스킵(되먹임/중복 저장 방지).
- 기억에 있으나 폴더가 사라진 경우 → 키는 남지만 무해(다음에 그 경로 재생성 시 적용). 정리 불필요(작은 dict).

## 6. 테스트 전략

- `Settings` 라운드트립: `AppSettings`에 libraryLayouts 채워 encode→decode 동일. **libraryLayouts 키 없는 JSON 디코드 → 빈 dict**(하위호환).
- AppState(@MainActor):
  - 기본값: libraryLayouts 빈 상태에서 selectedFolder 설정 → libraryLayout 변동 없음(미기억 유지).
  - 기억 복원: `settings.libraryLayouts["/v/photos"]=.list` 후 `selectedFolder=URL("/v/photos")` → `libraryLayout==.list`.
  - 저장: `selectedFolder=URL("/v/docs")` 후 `libraryLayout=.list` → `settings.libraryLayouts["/v/docs"]==.list`.
  - 복원이 저장을 유발하지 않음: 기억 복원 시 다른 폴더 키가 새로 안 생김(isRestoringLayout 동작).
- UI(툴바 토글·실제 복원 체감)는 빌드+수동.
